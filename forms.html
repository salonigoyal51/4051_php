Forms in PHP

Forms are a crucial part of web development that allow users to send data to the server. PHP is used to collect, validate, and process this data securely and efficiently.

A form typically includes input fields, checkboxes, radio buttons, and submit buttons. PHP retrieves the form data using superglobal variables like $_GET and $_POST.

Proper validation ensures that only correct and safe data is submitted, making form handling one of the most important aspects of PHP web development.

Forms in PHP are an essential part of web development. They allow users to enter data, which can then be sent to the server for processing. PHP handles form data sent through the HTML form using the GET or POST methods.

1. Creating an HTML Form:
An HTML form collects user input through input fields, radio buttons, checkboxes, dropdowns, and text areas. The form’s action attribute specifies the PHP file that will process the data, and the method attribute defines how the data will be sent (GET or POST).

Example:
<form action="process.php" method="post">
Name: <input type="text" name="username"><br>
Email: <input type="email" name="email"><br>
<input type="submit" value="Submit">
</form>

2. Accessing Form Data in PHP:
PHP provides two superglobal arrays to access submitted form data:
$_GET – Used when form method is GET.
$_POST – Used when form method is POST.
$_REQUEST – Can access data from both GET and POST methods.

Example (process.php):
<?php
$name = $_POST['username'];
$email = $_POST['email'];
echo "Welcome, $name! Your email is $email.";
?>

3. Difference between GET and POST Methods:
GET: Sends data through the URL. Useful for small, non-sensitive data. Data is visible in the browser’s address bar.
POST: Sends data through HTTP headers. More secure and used for sensitive information like passwords.

Example of GET Method:
<form action="welcome.php" method="get">
<input type="text" name="user">
<input type="submit">
</form>
The data will appear as: welcome.php?user=John

4. Validating Form Data:
Validation ensures that the user has entered correct and complete information. PHP allows both client-side (using JavaScript) and server-side validation. Common validation checks include empty fields, valid email format, and correct input type.

Example of Server-Side Validation:
<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
if (empty($_POST["username"])) { echo "Name is required."; }
else { echo "Hello, " . $_POST["username"]; }
}
?>

5. Using htmlspecialchars() Function:
The htmlspecialchars() function is used to prevent XSS (Cross-site scripting) attacks by converting special HTML characters into entities.
Example:
$name = htmlspecialchars($_POST['username']);

6. Self-Processing Form:
You can handle both the form display and processing in the same file using the PHP $_SERVER["PHP_SELF"] variable.
Example:
<form method="post" action="<?php echo $_SERVER['PHP_SELF']; ?>">
<input type="text" name="name">
<input type="submit">
</form>
<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
echo "Welcome " . $_POST["name"];
}
?>

7. Example Program:
<form method="post" action="process.php">
Name: <input type="text" name="name"><br>
Age: <input type="number" name="age"><br>
<input type="submit" value="Submit">
</form>

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
$name = $_POST["name"];
$age = $_POST["age"];
echo "Hello $name, you are $age years old.";
}
?>

Output:
Hello Saloni, you are 20 years old.

8. Summary:
Forms in PHP are used to collect and process user input. Data can be sent using GET or POST methods. PHP provides superglobal arrays like $_GET, $_POST, and $_REQUEST to access form data. Using functions like htmlspecialchars() and server-side validation helps make forms secure and reliable.

Example Programs

Program 1: Simple Form using GET Method

<form method="get" action="forms.php">
    Enter your name: <input type="text" name="username">
    <input type="submit" value="Submit">
</form>

<?php
if (isset($_GET['username'])) {
    echo "Welcome, " . $_GET['username'] . "!";
}
?>

Output:

Welcome, Saloni!

Program 2: Simple Form using POST Method

<form method="post" action="forms.php">
    Enter your city: <input type="text" name="city">
    <input type="submit" value="Submit">
</form>

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    echo "Your city is: " . $_POST['city'];
}
?>

Output:

Your city is: Chandigarh

Program 3: Validating Empty Form Fields

<form method="post" action="forms.php">
    Name: <input type="text" name="name">
    <input type="submit" value="Submit">
</form>

<?php
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    if (empty($_POST['name'])) {
        echo "Name is required!";
    } else {
        echo "Hello, " . $_POST['name'];
    }
}
?>

Output:

Name is required! or Hello, Saloni

Program 4: Radio Button Input

<form method="post" action="forms.php">
    Gender:
    <input type="radio" name="gender" value="Male">Male
    <input type="radio" name="gender" value="Female">Female
    <input type="submit" value="Submit">
</form>

<?php
if (isset($_POST['gender'])) {
    echo "Gender: " . $_POST['gender'];
}
?>

Output:

Gender: Female

Program 5: Dropdown List Example

<form method="post" action="forms.php">
    Select your country:
    <select name="country">
        <option value="India">India</option>
        <option value="USA">USA</option>
        <option value="UK">UK</option>
    </select>
    <input type="submit" value="Submit">
</form>

<?php
if (isset($_POST['country'])) {
    echo "You selected: " . $_POST['country'];
}
?>

Output:

You selected: India

Summary

Forms in PHP enable interaction between users and the server by collecting and processing input data. Using $_GET and $_POST methods, developers can access submitted information.

By validating form data, you ensure secure and accurate user input. Understanding form handling is essential for creating login pages, feedback forms, and registration systems in PHP.
